### Class Structure

1. Extend the generated abstract evaluator: `class MyEvaluator extends XxxEvaluator<ReturnType>`
2. Choose the return type `<T>`:
   - `<Double>` for a numeric calculator
   - `<Object>` for a multi-type language (numbers, strings, booleans)
   - `<String>` for a code emitter/transpiler
3. Add any state the evaluator needs (variable map, context object, etc.)

### BinaryExpr Pattern (for P4 AST with List fields)

When the generated AST uses the P4 pattern where `op` and `right` are Lists:

```java
record BinaryExpr(BinaryExpr left, List<String> op, List<BinaryExpr> right)
```

Handle three cases:

```java
@Override
protected Object evalBinaryExpr(BinaryExpr node) {
    BinaryExpr left = node.left();
    List<String> op = node.op();
    List<BinaryExpr> right = node.right();

    // 1. Leaf: literal value (left==null, op=[value], right=[])
    if (left == null && right.isEmpty() && op.size() == 1) {
        return parseLiteral(op.get(0));
    }
    // 2. Wrap: passthrough (left!=null, op=[], right=[])
    if (left != null && op.isEmpty() && right.isEmpty()) {
        return eval(left);
    }
    // 3. Binary: left op[0] right[0] op[1] right[1] ...
    Object current = eval(left);
    for (int i = 0; i < Math.min(op.size(), right.size()); i++) {
        Object r = eval(right.get(i));
        current = applyOp(op.get(i), current, r);
    }
    return current;
}
```

### BinaryExpr Pattern (for simple AST with scalar fields)

When the AST uses scalar fields:

```java
record BinaryExpr(ASTNode left, String op, ASTNode right)
```

```java
@Override
protected Double evalBinaryExpr(BinaryExpr node) {
    Double left = eval(node.left());
    Double right = eval(node.right());
    return switch (node.op()) {
        case "+" -> left + right;
        case "-" -> left - right;
        case "*" -> left * right;
        case "/" -> left / right;
        default -> throw new IllegalArgumentException("Unknown operator: " + node.op());
    };
}
```

### Variable Reference Pattern

```java
@Override
protected Object evalVariableRefExpr(VariableRefExpr node) {
    String name = node.name();
    // Strip $ prefix if present
    if (name.startsWith("$")) name = name.substring(1).strip();
    return context.getVariable(name);
}
```

### Conditional Pattern

```java
@Override
protected Object evalIfExpr(IfExpr node) {
    Boolean condition = (Boolean) eval(node.condition());
    return condition ? eval(node.thenExpr()) : eval(node.elseExpr());
}
```

### Passthrough/Wrapper Pattern

```java
@Override
protected Object evalExpressionExpr(ExpressionExpr node) {
    return eval(node.value());
}
```

### Literal Pattern

```java
@Override
protected Double evalNumberLiteral(NumberLiteral node) {
    return Double.parseDouble(node.value());
}
```

### Comparison Pattern

```java
@Override
protected Boolean evalComparisonExpr(ComparisonExpr node) {
    double left = ((Number) eval(node.left())).doubleValue();
    double right = ((Number) eval(node.right())).doubleValue();
    return switch (node.op()) {
        case "==" -> left == right;
        case "!=" -> left != right;
        case "<"  -> left < right;
        case ">"  -> left > right;
        case "<=" -> left <= right;
        case ">=" -> left >= right;
        default -> throw new IllegalArgumentException("Unknown comparator: " + node.op());
    };
}
```

### Function Call Pattern

```java
@Override
protected Double evalSinExpr(SinExpr node) {
    Double arg = (Double) eval(node.arg());
    return Math.sin(arg);
}

@Override
protected Double evalMinExpr(MinExpr node) {
    Double left = (Double) eval(node.left());
    Double right = (Double) eval(node.right());
    return Math.min(left, right);
}
```

### Checklist Before Returning

- [ ] Every `abstract` method from the skeleton is implemented
- [ ] BinaryExpr handles leaf/wrap/binary (or scalar left/op/right)
- [ ] Variable references strip `$` prefix if applicable
- [ ] Conditional expressions evaluate lazily (only evaluate the chosen branch)
- [ ] All operator strings match exactly what the grammar produces (e.g., `"+"`, not `"add"`)
- [ ] Type conversions are handled (Number -> double, etc.)
- [ ] Null safety for optional fields (check for null before eval)
